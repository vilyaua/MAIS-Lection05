# LangChain Document Loading and Text Splitting for RAG Pipelines

## Introduction / Overview
LangChain provides a comprehensive framework for implementing document loading and text splitting, which are foundational operations in Retrieval-Augmented Generation (RAG) pipelines. These steps are critical to transform raw documents from various formats into optimized chunks for embedding, searching, and retrieval by large language models (LLMs).

## Document Loading in LangChain
LangChain uses specialized **Document Loaders** to ingest raw data from different source types into a standardized `Document` object format. These loaders support a variety of formats such as plain text files, PDFs, web pages, and batch directories.

- **Document Object**: Contains `page_content` (text) and `metadata` (source, page number, file size, creation date, etc.). This consistent format supports downstream splitting, embedding, and retrieval.
- **Common Loaders**:
  - `TextLoader` for plain text files with encoding detection.
  - `PyPDFLoader` for PDF files, loading page-by-page and preserving page boundaries.
  - `WebBaseLoader` for fetching and cleaning web content.
  - `DirectoryLoader` for batch loading using glob patterns.
- **Loading Methods**:
  - `.load()` returns all documents at once (suitable for smaller files).
  - `.lazy_load()` returns a generator for memory-efficient incremental loading, essential for large files or many documents.

Example (simplified):
```python
from langchain_community.document_loaders import TextLoader
text_loader = TextLoader("sample_document.txt", encoding="utf-8")
docs = text_loader.load()
```

## Text Splitting in LangChain
Because many documents are too large for an LLM's context window, LangChain uses **Text Splitters** to break documents into smaller chunks suitable for indexing and retrieval.

- **RecursiveCharacterTextSplitter** is the recommended splitter for generic text, which recursively splits text using common delimiters (new lines, paragraphs) until chunks fit a target size.
- Parameters often include:
  - `chunk_size`: target length in characters.
  - `chunk_overlap`: to maintain context continuity between chunks.

Example (simplified):
```python
from langchain.text_splitter import RecursiveCharacterTextSplitter

splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(docs)
```

## Integration in a RAG Pipeline
The typical RAG document processing pipeline in LangChain follows:
1. **Load**: Use appropriate document loader to ingest data as a list or generator of `Document` objects.
2. **Split**: Use a text splitter to segment documents into manageable chunks.
3. **Embed & Store**: Embed chunks using an embedding model and store in a vector store.
4. **Retrieve & Generate**: At query time, retrieve relevant chunks from the vector store then generate answers using LLM.

Example snippet from documentation:
```python
loader = PyPDFLoader("document.pdf")
docs = loader.load()
splitter = RecursiveCharacterTextSplitter(chunk_size=1000, chunk_overlap=200)
chunks = splitter.split_documents(docs)
```
